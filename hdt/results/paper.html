Emergent Hierarchical Strategy Discovery in Pac-Man
via Gumbel-Softmax Decision Transformer

Anonymous Authors
Conference on Computer Vision and Pattern Recognition (CVPR) 2026
Abstract. We present a Hierarchical Decision Transformer (HDT) that discovers interpretable behavioral strategies in the classic arcade Pac-Man game through pure reinforcement learning, without any expert demonstrations or hand-crafted strategy labels. Our architecture combines a Transformer-based macro planner with an MLP-based micro executor, connected via 8 discrete latent codes sampled through Gumbel-Softmax. Trained with PPO on the original arcade maze (28×31 grid, 244 pellets, 4 ghosts) using a faithful juice-drop reward mapping from primate neuroscience experiments, the agent achieves 7/8 active latent codes with distinct behavioral signatures. Most notably, Code 2 emerges as a dedicated ghost-hunting mode (95.4% activation during frightened-ghost states, 7 kills with 0 deaths), while Code 7 develops into a dominant safe-foraging strategy (ghost distance 0.366, accounting for 68.3% of gameplay). The agent performs 3,757 strategy switches and 54 ghost kills across 100 evaluation episodes, all from a reward signal faithful to the original game rules.

1. Introduction

Hierarchical reinforcement learning (HRL) promises to decompose complex sequential decision-making into manageable sub-problems [1,2]. However, most HRL methods require manually defined subgoals, options, or intrinsic reward functions, limiting scalability and introducing human bias into the strategy space.

Recent work in primate neuroscience has demonstrated that macaque monkeys can learn multi-stage Pac-Man strategies through curriculum training with juice rewards [11]. Inspired by this paradigm, we investigate whether an artificial agent with a hierarchical information bottleneck can autonomously discover analogous behavioral strategies — foraging, evasion, and ghost-hunting — through pure reward-driven learning on the same game environment.

Our key contributions are:

(1) A dual-level architecture combining a Transformer planner (for long-horizon strategy selection) with an MLP executor (for reactive action control), connected via discrete Gumbel-Softmax latent codes.
(2) Faithful reproduction of the classic arcade Pac-Man maze (28×31, 244 pellets) with juice-drop-proportional rewards from primate experiments.
(3) Emergent discovery of 7 distinct behavioral strategies including a perfect hunter (7 kills, 0 deaths) and a safe forager (68% of gameplay, maximal ghost avoidance), without any strategy labels.
(4) Critical engineering insights: action masking, per-environment GAE, slow Gumbel temperature annealing, and code diversity regularization.

2. Related Work

Hierarchical RL. The Options framework [1] and Feudal Networks [2,3] establish foundations for temporal abstraction. HDQN [4] requires hand-crafted subgoals; Option-Critic [5] learns options end-to-end but suffers from option collapse. Our Gumbel-Softmax bottleneck with diversity loss addresses collapse directly.

Decision Transformers. Chen et al. [6] reframe RL as sequence modeling conditioned on returns. We apply Transformers specifically to the high-level planner for temporal context aggregation, while using a simpler MLP for reactive low-level control.

Emergent Behavior. DIAYN [7] learns diverse skills via mutual information maximization. VQ-VAE skill discovery [8] uses discrete bottlenecks for latent skill spaces. Our approach achieves similar emergent diversity within a fully on-policy PPO framework, using the game's native reward signal.

Primate Pac-Man. Yang et al. [11] trained macaque monkeys on Pac-Man through a 3-stage curriculum (reward collection, ghost interaction, energizer usage) with juice-drop rewards. We adopt their reward proportions and demonstrate that artificial agents discover analogous strategies.

4. Model Architecture

4.1 Overview

The HDT (Figure 3) consists of two components connected by a discrete latent code, with a temporal bottleneck constraining code update frequency.

4.2 High-Level Network (Transformer Planner)

A 2-layer Transformer encoder processes the last 16 frames of feature vectors. It outputs: (a) logits over 8 discrete strategy codes, sampled via Gumbel-Softmax [10] with straight-through gradient estimation; and (b) a scalar value estimate for the PPO critic. Architecture: hidden dim 64, 4 attention heads, sinusoidal positional encoding.

4.3 Low-Level Network (MLP Executor)

A 2-layer MLP (128 hidden units, Tanh activation) takes the current observation (42 dims) concatenated with the active code's one-hot vector (8 dims) = 50 input dims. It outputs action logits over 4 directions with action masking: invalid moves (walls) are set to $-10^8$ before sampling, ensuring the agent never attempts to walk into walls.

4.4 Temporal Bottleneck

The high-level network is queried every K=5 steps. Between queries, the same code persists. This forces the planner to make durable strategic commitments (e.g., "forage" vs. "hunt") rather than micro-managing each step, and enables the low-level executor to optimize within a stable strategic context.

Total model parameters: 94,094.

Figure 3. HDT Architecture — Transformer planner selects strategy codes every K=5 steps; MLP executor outputs masked actions at every step.

The Transformer planner aggregates 16-frame temporal context and emits a discrete strategy code. The MLP executor combines the current observation with the code one-hot vector and outputs wall-masked action logits. The environment loop feeds back the next observation.

5. Training

5.1 PPO with Dual-Level Optimization

Both levels are trained simultaneously with PPO [9]. The low-level loss follows standard PPO. The high-level loss includes additional terms for strategy exploration:

L_high = L_clip + 0.5*L_value + 0.20*L_entropy + 0.10*D_KL(p_code || Uniform(8))

The high entropy coefficient (0.20 vs. 0.02 for low-level) and code diversity KL loss jointly prevent mode collapse, encouraging all 8 codes to develop distinct usage patterns.

5.2 Gumbel-Softmax Temperature

The temperature anneals from 2.0 to 0.5 over 2M steps. This is deliberately slow — fast annealing (e.g., 100K steps) causes catastrophic mode collapse where only 2/8 codes remain active (see Section 7).

5.5 Implementation Details

Action Masking. Features [30:34] encode walkable directions. Invalid actions are masked to $-10^8$ before both sampling (training) and low-temperature sampling (evaluation, τ=0.3). Without this, greedy policies loop at walls indefinitely.

Per-Environment GAE. Generalized Advantage Estimation must be computed separately per environment. Interleaving transitions from multiple environments produces incorrect advantage estimates and prevents convergence.

Tunnel Wrapping. Side tunnels wrap horizontally, faithfully reproducing the arcade mechanic.

K=5 vs K=10: Reducing the temporal bottleneck from 10 to 5 steps nearly doubled ghost kills (29→54) and increased code switches by 2.7× (1,383→3,757). Faster strategy switching enables the agent to react to the 42-step frightened window before it expires.

Reward shaping removal: We initially included proximity-to-pellet bonuses and wall-bump penalties. These were removed to maintain scientific rigor: the final model trains on reward signals faithful to the original game rules. Training converges without shaping, demonstrating that the architecture is sufficient for strategy emergence.

8. Discussion

8.1 Emergent Behaviors

Ghost Hunting (Code 2): Activates when 95.4% of ghosts are frightened, achieves a perfect 7-kill-0-death record. This multi-step behavior (navigate to energizer → consume → pursue frightened ghost → eat) emerged purely from the +8.0 reward signal, demonstrating that architectural inductive bias (temporal bottleneck + discrete codes) enables complex sequential strategy discovery.

Safe Foraging (Code 7): Maintains the highest ghost distance (0.366) among all codes, accounts for 68.3% of total gameplay, and dominates the late-game phase (180 continuous steps in Figure 6b). This suggests the model learned that survival is the dominant long-term strategy.

Active Foraging (Code 4): Despite lower ghost distance (0.276), this code eats the most pellets (3,396) — it operates in moderate-risk zones where pellet density is higher. The behavioral distinction between Codes 4 and 7 mirrors the explore-exploit tradeoff.

8.2 Comparison with Primate Strategies

Yang et al. [11] observed three distinct behavioral phases in monkeys: (1) pellet collection, (2) ghost avoidance, (3) energizer utilization with ghost hunting. Our model's emergent codes parallel this hierarchy: Codes 4/7 correspond to phase 1, Code 1's ghost-distance sensitivity reflects phase 2, and Code 2 directly implements phase 3.

8.3 Limitations

(1) No dedicated evasion code emerged; ghost avoidance is encoded implicitly via distance-sensitive codes. (2) 30% pellet ratio leaves room for improvement. (3) The MLP executor limits multi-step path planning. (4) Code 2 (Hunter) is activated only 0.5% of the time, suggesting the frightened window is underutilized.

9. Conclusion

We demonstrate that a lightweight hierarchical architecture (94K parameters) with a discrete Gumbel-Softmax bottleneck discovers interpretable Pac-Man strategies through pure reinforcement learning on the classic arcade maze with faithful game-rule rewards. The emergent ghost-hunting code (95.4% frightened-state activation, 7 kills, 0 deaths) represents complex multi-step planning that arose without any hunting-related supervision. The 3,757 strategy switches across 100 episodes demonstrate active, context-dependent policy composition. These results suggest that architectural inductive biases — temporal bottleneck, discrete information bottleneck, and code diversity pressure — are sufficient to elicit emergent hierarchical behavior in complex sequential decision tasks.
